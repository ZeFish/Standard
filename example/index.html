Logical vs Physical Properties

The Standard Framework uses logical properties for international typography support:

‚úÖ Logical Properties (Framework Default)
/* Adapts to writing modes automatically */
.rhythm > * {
  margin-block-end: var(--space);
}

.reading {
  padding-inline: var(--space-l);
}

Works in LTR, RTL, and vertical writing modes

‚ùå Physical Properties (Avoid)

  margin-bottom: var(--space);
}

.old-prose {
  padding-left: var(--space-l);
}

Breaks in RTL languages and vertical text

Reading Layout Demonstration

The main text sits in the default content area, following optimal reading width principles.

Small Area

Narrower than content, perfect for side notes or supplementary information.

Accent Area

Wider than content, good for pullquotes or highlighted content.

Feature Area

Even wider, perfect for images, tables, or featured content blocks.

Full Width

Extends to the full container width, like full-bleed images or background sections.

Layout Area Reference

.reading: Main layout container with flexible content areas
.content: Default content area with optimal reading width
.small: Narrower area, --space smaller on each side
.accent: Wider area for pullquotes and highlights
.feature: Even wider for images and featured content
.full: Full container width for backgrounds and bleeds
